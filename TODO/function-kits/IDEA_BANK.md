# Function Kit 场景与选题库（Idea Bank）

> 编码：UTF-8  
> 用途：从“灵感”收敛到“可落地的下一件功能件”。  
> 核心材料：`product-thinking/doubao-thread-wce998b8e58c59ca0.md`（含大量场景、约束与宣发方案）。  

## 0. 选题筛选标准（不满足就先别做）

强制 4 条：

1. **输入法独有**：必须利用「表达链路/发送前/剪贴板闭环/宿主场景识别/输入过程上下文」之一。
2. **高频**：要么每天多次用，要么一用就强传播（破圈）。
3. **低风险**：不碰灰产、不依赖脆弱爬虫/封号风险，不做“偷看别的 App 数据”的功能。
4. **1-1-1**：1 步唤起、1 步执行、1 步返回。

## 1. 我建议优先考虑的 3 个“尖刀级”方向（更像输入法专属）

### A. Clipboard 闭环：复制→处理→一键插入/替换

- 为什么值得：最符合“零切换”，且隐私边界清晰（用户主动复制）。
- 典型入口：复制后 chip（用户点）/ `/粘贴` / 选中菜单
- 需要的能力：`context.read` + `input.insert|replace` +（可选）`network.fetch` / `ai.request`

候选功能件：

1. **格式清理 + 规范化粘贴**（纯本地优先）  
   - 爽点：替代“先粘贴到记事本清格式”的肌肉记忆  
   - 风险：低（本地处理）  
2. **结构化提取粘贴**（地址/电话/时间/清单）  
   - 爽点：复制一段杂乱信息，自动拆字段/填表（仍要用户确认写回）  
   - 风险：中（需要输入框类型识别；保守策略）  
3. **翻译/总结/润色后粘贴**（异步，不阻塞输入）  
   - 爽点：复制长文→一键总结成 3 点并插入  
   - 风险：中（云端/费用/隐私提示与降级）  

### B. Send intercept：发送前最后一道防线（克制但强价值）

- 为什么值得：独立 App 很难卡住“发送瞬间”，输入法是天然的最后一米。
- 典型入口：发送动作拦截 + 二次确认
- 需要的能力：`send.intercept.ime_action` +（可选）`ai.request`

候选功能件：

1. **敏感信息检查**（手机号/证件/金额单位/隐私泄露）  
2. **情绪上头冷静/语气降温**（提示 + 一键替换 + 可忽略）  
3. **合规/平台限流风险提示**（可选；更偏运营/营销人群，默认不要作为第一爆款）  

### C. 短文本场景化改写：同一段话“一键变得能直接发出去”

- 为什么值得：核心优势不是“写得多专业”，而是**不复制、不切 App**，在输入框里把一句话变成“更好发的那一句”。
- 典型入口：`/改写`、`/更礼貌`、`/更短`、`/更清楚`、工具栏按钮
- 需要的能力：`context.read` + `input.replace` +（可选）`ai.request`

候选功能件：

1. **聊天/沟通一键改成 3 种语气**（更礼貌 / 更坚定 / 更简短；用户点一下替换）  
2. **把“想法碎片”改成“可发送的一句话”**（把口语/碎句整理成通顺表达）  
3. **把“问题”改成“更好被回答/被搜索的提问”**（补全信息、去歧义、列出缺失条件）  

> 说明：像“小红书/抖音发长文格式化”这类 **发布型长内容**，真实行为往往是“在专业编辑器写好再粘贴发布”，不适合作为第一批核心功能件；最多作为“短评论/回复”的轻量能力存在。

## 2. 其他高潜但需要更谨慎的方向

### 社交沟通类（高频、易传播，但隐私与平台规则要更克制）

1. **聊天人设一键适配**：按联系人/群聊选择语气模板，生成 3 个可选回复  
2. **聊天记忆锚点**：本地提取“生日/忌口/约定”等可显式维护的信息（不要默认抓取全量聊天记录）  
3. **圈层黑话双向转译**：接收释义 + 发送改写（默认手动触发）  

### 动态监听类（默认关闭，只做“主动触发做不到”的真痛点）

1. **反复改句子→给一个极小提示**：用户点击才展开“语气微调/更专业/更简短”  
2. **发送前强风险才拦截**：低风险只提示可忽略，高风险才二次确认  

> 原则：白名单 + 频率上限 + 可解释 + 可一键永久关闭。

## 3. 每个候选功能件的落地问卷（10 分钟出结论）

对任意候选，回答下面 10 个问题；答不出来就先别写代码：

1. 一句话价值是什么？（用户愿意换输入法的理由）
2. 入口是什么？能不能 1 步唤起？
3. 默认执行是什么？能不能 1 步完成？
4. 用完怎么回到输入？有没有阻塞输入？
5. 需要哪些权限？能不能更少？
6. 数据边界是什么？哪些数据绝不触碰？
7. 失败怎么降级？离线/弱网怎么办？
8. 对比“复制到 AI 工具里做”，你多出来的体验优势是什么？
9. 成功指标怎么量？（触发次数、转化、留存、关闭率）
10. 宣发钩子是什么？（用户会不会主动分享/转发）
