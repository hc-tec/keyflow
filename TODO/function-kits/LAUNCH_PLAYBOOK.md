# Function Kit 宣发与增长流程（Launch Playbook）

> 编码：UTF-8  
> 目标：把“做出来”变成“有人用、会传播、能持续”。  
> 主要来源：`product-thinking/doubao-thread-wce998b8e58c59ca0.md`（宣发全周期方案 + 风险与复盘）。  

## 1. 定位：别把自己讲成“又一款输入法”

更有效的叙事：

- **可插拔功能件输入法平台**
- 对标心智：**输入法界的 Chrome 应用商店 / 万能插件中心**

核心公式：

**单爆款功能件尖刀引流 → 输入法本体承接 → 生态能力留存**

## 2. 人群分层：不同人群用不同尖刀打

建议的四层人群（用于挑“第一批官方功能件”）：

- 种子用户：技术极客/效率工具爱好者/开源用户（隐私、可控、极致效率）
- 核心增长：自媒体/运营/职场（文案、合规、提效）
- 泛流量：Z 世代/社交活跃（好玩、情绪价值、愿意分享）
- 生态用户：开发者（SDK、分发、变现预期）

## 3. 分阶段策略（不要一上来就铺大）

### 阶段 1：冷启动（0-10 万用户）

目标：验证 PMF，跑通“功能件引流→下载→留存”闭环。

- 只打磨 3 个尖刀功能件（各击穿一类人群）
- 渠道：开发者/效率社区 + 私域社群 + 中腰部内容合作
- 内容：每条内容只讲 1 个功能件、1 个痛点、1 个“1 步闭环”的爽点

### 阶段 2：破圈增长（10-100 万用户）

目标：用趣味功能件破圈，用效率功能件留存，规模化做内容矩阵。

### 阶段 3：生态成熟（100 万+）

目标：开放第三方功能件、商店、审核、评分、下架、分成机制，形成正向循环。

## 4. 宣发“Skills”体系（提示词工程的资产化）

把渠道经验固化成可复用模板（可用于你未来的“宣发后台”）：

- 社区爆款笔记模板
- 短视频口播脚本模板
- 知乎深度种草文模板
- 开发者社区推广文模板
- 社群裂变话术模板

实践方法：

- 固化：把变量抽成占位符（功能件/人群/爽点/合规边界）
- 迭代：每次投放记录数据（曝光、点击、下载、留存），反哺模板

## 5. 风险与复盘（必须提前设计）

- 合规：避免绝对化用语；隐私与权限必须可解释、可关闭
- 平台限流：用“真实体验分享”而不是硬广；引导下载要克制
- 舆情：重点监控“隐私/卡顿/功能失效”；快速响应

最小复盘指标（建议先做到可观测）：

- 触发/使用：功能件安装率、使用率、执行成功率
- 结果：insert/replace 转化率、满意/重试/取消
- 留存：次日/7 日留存，功能件复用率
- 负反馈：关闭率、永久关闭率、卸载前行为
