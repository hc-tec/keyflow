# 05｜复制长文 → 一键总结后粘贴（3 点 + 1 句）

> 编码：UTF-8  
> 状态：调研草案  
> 关键词：Summarize / TL;DR / 复制即总结  

## 1. 一句话价值

复制一段长内容（公告/对话/文章），在输入框里一键得到“**3 条要点 + 1 句结论**”，直接替换/插入发送。

## 2. 典型场景

1. 同事发了一段很长说明，你想“快速转述给另一个人”  
2. 把一段公告发到群里，但希望先压缩成要点  
3. 把一段聊天记录摘要发给领导/老师（不需要原文细节）  

## 3. 为什么输入法独有

- 独立 AI 工具也能总结，但用户真实行为是：**嫌麻烦不去用**。  
- 输入法的优势是把这件事变成“复制后的一步”，并且结果**直接写回当前输入框**。

## 4. 交互设计（1-1-1）

### 4.1 入口

- 复制后 chip：当剪贴板文本长度超过阈值（例如 > 300 字）显示“总结后粘贴”

### 4.2 默认执行

默认输出结构（可复制粘贴也可直接替换）：

- 要点 1
- 要点 2
- 要点 3
结论：一句话

按钮：replace / insert。

### 4.3 返回

异步执行不阻塞输入；完成后以低打扰方式提示“可插入总结”，用户点一次写回。

## 5. 权限与数据边界（必须把信任讲清楚）

### 5.1 最小权限

- `ai.request`（或 `ai.agent.run`，但建议先用 request）
- `input.insert` / `input.replace`
- （可选）`tasks`：让 UI 有“进行中/可取消/历史”

### 5.2 关键边界

- 默认不自动上传：必须用户点击触发才发送
- UI 必须明确提示：内容将发送到“已配置的 AI 后端”（或本地模型）
- 失败降级：AI 不可用时，提示“仅做本地压缩（截断/去空行）”

## 6. MVP 范围

**Must**

- 复制后 chip 触发
- AI 总结（3 点 + 1 句）
- insert/replace
- 可取消

**Should**

- 三种长度：超短/标准/详细（一个 segmented 控件）

**Won’t**

- 自动识别“对谁总结/什么语气”（先不做个性化）

## 7. 竞品与参考

- ChatGPT 等通用工具的“复制→粘贴→总结”路径（对比用来证明“零切换”价值）
- 现有 IME/邮箱中的“Smart Reply/Smart Compose”可以作为“低打扰异步建议”的交互参考：  
  - Gmail Smart Reply：`https://support.google.com/mail/answer/9116836`  

## 8. 风险与缓解

- **隐私顾虑**：必须显式告知 + 默认关闭自动上传 + 支持本地/自托管后端。  
- **结果不稳定**：提供“换一种总结”“更短/更长”快捷按钮。  
- **卡顿/阻塞输入**：必须异步 + tasks 化，不允许锁住输入。  

## 9. 指标

- 触发→执行率、完成率
- 结果写回率（insert/replace）
- 平均耗时、取消率
- 复用率（7 日内使用次数）

## 10. 工程备注

- UI 调用：`await kit.ai.request({ ... })` 获取结构化结果
- 用 tasks 承载“进行中/历史”，避免用户觉得“点了没反应”
