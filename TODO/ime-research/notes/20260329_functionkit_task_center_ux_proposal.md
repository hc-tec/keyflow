# 2026-03-29 Function Kit「任务中心」入口与内容改造方案

## 背景 / 现状

当前 Android IME 的「任务中心」入口在三级路径里：

1. 打开工具栏（滑动/点展开）
2. 点击「更多」
3. 点击「任务中心」

问题：

- **可达性差**：当用户需要“立刻查看/取消/确认”任务进度时，三级路径会直接把用户劝退。
- **可发现性差**：入口藏在「更多」里，且没有“有任务在跑”的明显反馈。
- **信息过于开发者向**：列表标题展示 `kind/status`（如 `network.fetch (succeeded)`），详情默认展示整段 JSON + 复制 JSON，普通用户看不懂也不关心。

## 用户在任务中心真正想做什么

- **我现在有什么任务在跑？**（上传、网络请求、AI 请求、后台渲染等）
- **它卡住了吗 / 失败了吗？为什么？**（一句话可理解的原因）
- **我能做什么？**（取消/重试/回到对应功能件）
- **我怎么回到输入继续用？**（保持“键盘任务流”）

## 目标

- 把「任务中心」从“埋点功能”变成“可用的用户工具”。
- 入口做到 **一眼可见/一键可达**（至少不再进「更多」才找得到）。
- 信息做到 **默认用户可读**，开发者信息 **可选展开**。

## 方案（入口放置）

### 方案 A（推荐）：工具栏增加「任务中心」固定按钮

- 在工具栏（与剪贴板/光标/撤销/更多同级）增加一个「任务中心」按钮，点击直接打开任务中心窗口。
- 当存在 `running` 任务时：
  - 按钮显示 **红点/数字 badge**（后续增强项），让用户知道“有事在跑”。

优点：最直观、最少步骤（展开工具栏后 1 次点击）。
缺点：工具栏多一个按钮，占位。

### 方案 B：在「更多」按钮上做“短按=更多 / 长按=任务中心”

优点：不增加按钮数量。
缺点：长按不够可发现；对普通用户不友好，无法满足“立刻触及到”的诉求。

### 方案 C：任务开始时弹出轻提示（snackbar/气泡）提供「查看任务」

优点：任务发生时“主动引导”，用户不必自己找入口。
缺点：实现需要更谨慎（避免打扰、避免频繁弹）。

结论：**先落地方案 A**（保证可达性），再迭代方案 C（提升可发现性），方案 B 作为补充快捷手势而非主入口。

## 方案（内容改造）

### 1) 任务列表默认展示“用户语言”

当前：`kind (status)` + `kit/surface - summary`

建议改为：

- **主标题**：`任务标题`（对用户可读）
  - 优先用 `progress.message` / `result.summary` / `error.message`（这些字段最接近用户语言）
  - 若为空，再回退到 kind 映射（例如 `network.fetch` -> “网络请求”，`ai.request` -> “AI 请求”）
- **副标题**：`功能件名称` + `状态/时间`
  - 状态映射：queued/running/canceling/succeeded/failed/canceled -> 对应中文
  - 时间：显示相对时间或更新时间（后续增强项；第一步可先不做）

### 2) 任务详情默认“用户视图”，开发者信息折叠

详情页建议分成两层：

- **用户视图（默认）**
  - 一句话状态（进行中/成功/失败/已取消）
  - 最近的 progress message / error message
  - 按钮：取消（如可取消）、打开对应功能件、返回任务中心
- **开发者视图（可选）**
  - 展开后才显示 JSON
  - “复制 JSON”移到开发者视图里

### 3) 让功能件更容易提供“用户级摘要”

Host 侧已能从 task 中提取：

- `progress.message / progress.stage`
- `result.summary / result.message`
- `error.message / error.code`

建议在 Function Kit SDK/文档中明确：

- `progress.message` 应该是用户能看懂的话（例如“正在上传 2/5…”）
- `result.summary` 用作历史列表的短摘要（例如“上传完成”/“HTTP 200”）

## 分阶段落地计划

### Phase 1（立刻提升可达性）

- 工具栏新增「任务中心」按钮（固定入口）。
- 任务中心列表做最小的文案优化：kind/status 映射为用户语言（保留原字段作为兜底）。

### Phase 2（提升可发现性）

- 当存在 running 任务时，在工具栏按钮显示红点/badge。
- 任务开始/失败时弹出轻提示（带“查看任务”）。

### Phase 3（内容结构升级）

- 任务详情页改为“用户视图默认 + JSON 折叠”。
- 支持按功能件筛选/仅看失败/仅看进行中等。
