# 06｜翻译后粘贴（带语气）

> 编码：UTF-8  
> 状态：调研草案  
> 关键词：Translate / 本地化 / 口语化 / 商务化  

## 1. 一句话价值

复制一段文字，一键翻译成目标语言，并按“更口语/更正式/更简短”输出，直接插入/替换当前输入框。

## 2. 典型场景（非专业也高频）

1. 给朋友/同事发英文短消息（或反过来）  
2. 在海外 App/网站的输入框里回一句英文评论  
3. 把外语内容转述给家人/同学（中文更顺）  

## 3. 为什么输入法独有

- 翻译本身不独有，但输入法能做到**“复制→翻译→直接回到当前输入框”**，用户不切 App。  
- 相比通用翻译 App，输入法更接近“我马上要发出去的那句话”，因此“语气档位”更有价值（口语/礼貌/简短）。

## 4. 交互设计（1-1-1）

### 4.1 入口

- 复制后 chip：检测剪贴板语言与当前输入法语言不一致时出现“翻译并粘贴”
- `/翻译`：手动触发

### 4.2 默认执行

默认 UI：

- 目标语言：自动（基于 UI 语言/用户设置）+ 下拉可选
- 语气档位（segmented）：`口语` / `礼貌` / `简短`
- 输出预览（前 200 字）
- 按钮：insert / replace

### 4.3 返回

翻译异步执行，不阻塞输入；完成后用户一键写回。

## 5. 权限与数据边界

### 5.1 最小权限

- `ai.request`（推荐走你已跑通的 AI 能力）
- `input.insert` / `input.replace`
- `storage.write`（保存默认目标语言与语气档位）

### 5.2 边界与解释

- 明确提示“翻译内容将发送到已配置的 AI/翻译后端”（可选提供“离线模式：仅做本地替换规则”，但质量有限）
- 默认不记录历史；如要做历史，必须本地并可一键清空

## 6. MVP 范围

**Must**

- 复制后 chip 触发
- 目标语言自动 + 手动切换
- 3 档语气（提示词模板即可）
- insert/replace

**Should**

- “保持专有名词不翻译”开关（减少误译）
- “保留原文在下方”模式（适合对照）

**Won’t**

- 多轮对话翻译（容易变成重产品）

## 7. 竞品与参考（借鉴交互，不照搬）

- Google Translate（典型“切 App”的对照）：`https://translate.google.com/`
- DeepL（高质量翻译与语气相关心智）：`https://www.deepl.com/translator`

## 8. 风险与缓解

- **隐私顾虑**：默认手动触发、明确提示后端、提供“本地优先”说明。  
- **误译/语气不当**：提供“更口语/更正式/更简短”三档一键重试。  
- **成本**：对长文本做长度上限与提示，避免“复制整篇文章”烧钱。  

## 9. 指标

- 触发→执行率、写回率
- 语气切换次数（反映默认质量）
- 取消/重试率、平均耗时

## 10. 工程备注

- 推荐把“语气档位”实现为固定 prompt 模板（可复用）
- 输出尽量结构化：`{lang, tone, text}` 方便 UI 展示
